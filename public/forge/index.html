The flywheel

Engineering that compounds.

The killer feature. Every solved problem is captured as a typed learning and a linked note in an Obsidian memory vault. Before the next plan, the learnings-researcher recalls what's relevant — so the team sharpens with every task instead of starting from zero.

Durable memory

An Obsidian vault is the long-term brain.

Beyond the per-task docs/ handoff, lessons land in a linked Obsidian knowledge base reached over MCP — decisions-in-context, codebase gotchas, and the fixes for them, growing into a navigable graph the whole team draws on.

  • Linked notes, not a log. Every learning connects to the files, decisions, and prior lessons it relates to.
  • Dual-write on /learn. One typed artifact in docs/learnings/ and one frontmatter-safe vault note.
  • Recalled before you plan. learnings-researcher searches the vault first, so past work informs the next move.

The split

Plan heavy. Edit cheap.

A high-reasoning architect designs against a story; a fast implementer applies the diff. Right model, right job — better output, lower cost. Judgment stays in the main thread; context-heavy work is delegated.

Quality gates

Nothing ships on a vibe.

Verification is structural, not optional. Reviews fan out in parallel; completion claims require fresh evidence; disciplines load as enforced skills.

Parallel review fan-out

/review runs code, security, and performance auditors at once, then aggregates by severity — not by reviewer. Anti-slop guardrails kill confident-but-wrong findings.

Verification gate

/ship won't claim "done" without running the command and reading the output. Stub scan, real test results, every acceptance criterion checked.

Disciplines as skills

test-driven-development and systematic-debugging load on trigger — RED→GREEN→REFACTOR, and no fix without a reproduction and a named root cause.

The advantage

Why teams reach for it.

Not more AI — AI with the operating discipline of a good engineering org, encoded once and applied every session.

Survives /clear

State lives in git-tracked artifacts and a memory vault — not a fragile transcript. Reset anytime; pick up cold.

Compounds over time

The learning flywheel means task N+1 is informed by task N. Your AI team accrues institutional memory.

Right-sized, always

Complexity sets the depth, not a fixed process. Trivial work stays fast; risky work gets gated.

Anti-hallucination, anti-slop

A strict knowledge-verification chain and reviewer guardrails catch invented APIs and silent failures before they merge.

Cost-aware by design

Heavy model to plan, fast model to edit, parallel agents for breadth. Spend reasoning where it pays off.
